# 📝 قابلیت یادداشت‌های ربات

## توضیحات

قابلیت یادداشت به شما امکان می‌دهد تا برای هر ربات که با آن کار می‌کنید، یادداشت‌های شخصی ثبت کنید.

## ویژگی‌ها

✅ **ثبت خودکار**: بعد از اجرای هر سناریو، از شما پرسیده می‌شود آیا می‌خواهید یادداشت ثبت کنید
✅ **چند ربات**: برای سناریوهای چند ربات، برای هر ربات جداگانه یادداشت ثبت کنید
✅ **مدیریت کامل**: مشاهده، ویرایش و حذف یادداشت‌ها
✅ **جستجو**: مشاهده یادداشت‌های یک ربات خاص
✅ **ذخیره سناریو**: متن سناریو همراه با یادداشت ذخیره می‌شود

## نحوه استفاده

### 1. ثبت یادداشت بعد از سناریو

بعد از اجرای موفق سناریو، ربات از شما می‌پرسد:

```
📝 یادداشت برای ربات @BotName

آیا می‌خواهید یادداشتی برای این ربات ثبت کنید؟

💡 یادداشت‌ها به شما کمک می‌کنند تا اطلاعات مهم درباره ربات را ذخیره کنید.
```

**گزینه‌ها:**
- ✅ **بله، یادداشت می‌زنم** - برای ثبت یادداشت
- ⏭ **بعدی** - رد کردن و رفتن به ربات بعدی (فقط برای چند ربات)
- ❌ **نه، نیازی نیست** - عدم ثبت یادداشت

### 2. مشاهده یادداشت‌ها

#### از منوی اصلی:
کلیک روی دکمه **📝 یادداشت‌های من**

#### با دستور:
```
/notes @bot_username
```

مثال:
```
/notes @MyReferralBot
```

### 3. حذف یادداشت

```
/deletenote NOTE_ID
```

مثال:
```
/deletenote 5
```

💡 **نکته:** ID یادداشت را از دستور `/notes @bot_username` دریافت کنید.

### 4. ویرایش یادداشت

```
/editnote NOTE_ID
```

مثال:
```
/editnote 5
```

سپس متن جدید را ارسال کنید.

## مثال‌های کاربردی

### مثال 1: ثبت لینک رفرال

```
لینک رفرال: https://t.me/MyBot?start=ref_123456
تاریخ: 2024-01-15
نتیجه: 50 نفر جوین شدند
```

### مثال 2: ثبت تنظیمات

```
تنظیمات ربات:
- کانال اجباری: @MyChannel
- دکمه کلیک: "عضویت"
- زمان انتظار: 3 ثانیه
```

### مثال 3: ثبت نتایج

```
نتایج اجرا:
- تعداد اکانت: 100
- موفق: 95
- ناموفق: 5
- مشکل: کپچا در 3 اکانت
```

### مثال 4: ثبت نکات مهم

```
⚠️ نکات مهم:
- این ربات کپچا دارد
- باید از solve_captcha: send استفاده کرد
- بعد از جوین 5 ثانیه صبر کن
```

## ساختار دیتابیس

یادداشت‌ها در جدول `bot_notes` ذخیره می‌شوند:

```sql
CREATE TABLE bot_notes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    bot_username TEXT NOT NULL,
    note_text TEXT NOT NULL,
    scenario_text TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (user_id)
)
```

## دستورات

| دستور | توضیحات | مثال |
|-------|---------|------|
| `/notes @bot` | مشاهده یادداشت‌های یک ربات | `/notes @MyBot` |
| `/deletenote ID` | حذف یادداشت | `/deletenote 5` |
| `/editnote ID` | ویرایش یادداشت | `/editnote 5` |

## نکات مهم

1. **خصوصی**: هر کاربر فقط یادداشت‌های خودش را می‌بیند
2. **نامحدود**: تعداد یادداشت‌ها محدودیت ندارد
3. **جستجو**: می‌توانید یادداشت‌های یک ربات خاص را ببینید
4. **تاریخ**: تاریخ ایجاد و ویرایش ذخیره می‌شود
5. **سناریو**: متن سناریو همراه با یادداشت ذخیره می‌شود

## مزایا

✅ **سازماندهی**: اطلاعات رباتها را منظم نگه دارید
✅ **یادآوری**: نکات مهم را فراموش نکنید
✅ **تاریخچه**: نتایج قبلی را ببینید
✅ **اشتراک**: با تیم خود اطلاعات را به اشتراک بگذارید (اگر ادمین باشند)
✅ **بهینه‌سازی**: سناریوها را بر اساس نتایج قبلی بهبود دهید

## سوالات متداول

### چند یادداشت می‌توانم برای یک ربات داشته باشم؟
نامحدود! می‌توانید هر تعداد یادداشت که می‌خواهید ثبت کنید.

### آیا یادداشت‌ها برای همه قابل مشاهده است؟
خیر، هر کاربر فقط یادداشت‌های خودش را می‌بیند.

### آیا می‌توانم یادداشت قدیمی را ویرایش کنم؟
بله، با دستور `/editnote NOTE_ID` می‌توانید یادداشت را ویرایش کنید.

### آیا یادداشت‌ها با بکاپ ذخیره می‌شوند؟
بله، یادداشت‌ها در دیتابیس ذخیره می‌شوند و با بکاپ دیتابیس ذخیره می‌شوند.

## پشتیبانی

اگر سوالی دارید یا به مشکلی برخوردید، با ادمین تماس بگیرید.
