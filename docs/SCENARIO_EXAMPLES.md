# 🎯 راهنمای سناریو پیشرفته

## توضیحات کلی

سناریو پیشرفته به شما امکان می‌دهد یک فرآیند کامل تعامل با ربات‌های تلگرام را به صورت خودکار اجرا کنید.

## دستورات موجود

### 1. `start: ref_id`
استارت ربات با پارامتر رفرال

```
start: AAAAACWiOtQ
```

### 2. `send: متن`
ارسال پیام به ربات

```
send: سلام من کاربر جدید هستم
```

### 3. `click: کلمه کلیدی` یا `click: #N`
کلیک روی دکمه

**روش 1: جستجوی متن (برای دکمه‌های ثابت)**
```
click: عضویت
click: تایید
```

**روش 2: شماره دکمه (برای دکمه‌های متغیر)**
```
click: #0    # اولین دکمه
click: #1    # دومین دکمه
click: 2     # سومین دکمه (بدون # هم کار می‌کند)
```

💡 **نکته:** شماره دکمه‌ها از 0 شروع می‌شود:
- دکمه اول = 0
- دکمه دوم = 1
- دکمه سوم = 2
- و غیره...

### 4. `join: لینک`
جوین کانال یا گروه

```
join: https://t.me/mychannel
join: @mychannel
```

### 5. `leave: لینک`
لفت کانال یا گروه

```
leave: https://t.me/mychannel
leave: @mychannel
```

### 6. `wait: ثانیه`
صبر کردن

```
wait: 5
```

### 7. `forward: N, @target`
فوروارد N تا از آخرین پیام‌های ربات به کانال/گروه

```
forward: 5, @mychannel        # فوروارد 5 پیام آخر
forward: 3, @results_group    # فوروارد 3 پیام آخر
forward: 1, mychannel         # فوروارد 1 پیام آخر (بدون @ هم کار می‌کند)
```

💡 **نکته:** معمولاً در آخر سناریو استفاده می‌شود تا نتایج را جمع‌آوری کنید.

### 8. `#` کامنت
خطوط کامنت (نادیده گرفته می‌شود)

```
# این یک کامنت است
```

## متغیرهای دینامیک

### `{random:N}`
رشته تصادفی N حرفی (حروف کوچک + اعداد)

```
send: یوزرنیم من {random:8} است
```
خروجی برای اکانت 1: `یوزرنیم من abc12xyz است`
خروجی برای اکانت 2: `یوزرنیم من def34uvw است`

### `{random_upper:N}`
رشته تصادفی N حرفی (حروف بزرگ + اعداد)

```
send: کد من {random_upper:6} است
```
خروجی برای اکانت 1: `کد من ABC123 است`
خروجی برای اکانت 2: `کد من XYZ789 است`

### `{random_num:N}`
عدد تصادفی N رقمی

```
send: شماره من {random_num:4} است
```
خروجی برای اکانت 1: `شماره من 1234 است`
خروجی برای اکانت 2: `شماره من 5678 است`

## مثال‌های کامل

### مثال 1: ربات رفرال ساده

```
@Startraygannetbot
start: AAAAACWiOtQ
click: عضویت
join: https://t.me/mychannel
click: تایید
```

### مثال 2: ربات با ارسال یوزرنیم یونیک

```
@UsernameBot
start:
send: /register
send: یوزرنیم من {random:8} است
click: ثبت نام
```

### مثال 3: ربات با چند مرحله

```
@ComplexBot
start: ref123
click: شروع
send: نام من {random:5} است
wait: 3
click: ادامه
send: کد من {random_num:6} است
click: تایید نهایی
```

### مثال 4: ربات با جوین چند کانال

```
@MultiChannelBot
start:
click: عضویت
join: https://t.me/channel1
wait: 2
join: https://t.me/channel2
wait: 2
join: https://t.me/channel3
click: بررسی عضویت
```

### مثال 6: استفاده از شماره دکمه (برای دکمه‌های متغیر)

```
@DynamicBot
start: ref123
click: #0    # کلیک اولین دکمه (هر چی باشه)
wait: 2
click: 1     # کلیک دومین دکمه
send: یوزرنیم من {random:8} است
click: #2    # کلیک سومین دکمه
```

💡 **کاربرد:** وقتی متن دکمه‌ها برای هر اکانت متفاوت است، از شماره دکمه استفاده کنید.

### مثال 8: فوروارد نتایج به کانال

```
@RewardBot
start: ref123
click: دریافت جایزه
send: {random:8}
click: #0
# حالا نتیجه رو فوروارد می‌کنیم
forward: 2, @my_results_channel
```

💡 **کاربرد:** جمع‌آوری نتایج همه اکانت‌ها در یک کانال

### مثال 9: سناریو کامل با همه قابلیت‌ها

```
@CompleteBot
start: ref_abc123
click: شروع
send: نام من {random:5} است
click: #0
wait: 3
join: https://t.me/channel1
send: یوزرنیم: {random:8}
send: کد: {random_num:6}
click: 1
# فوروارد 5 پیام آخر به کانال نتایج
forward: 5, @results_channel
```

## نکات مهم

1. **خط اول باید یوزرنیم ربات باشد** (با @ یا بدون @)
2. **متغیرها برای هر اکانت متفاوت هستند** - هر بار که سناریو اجرا می‌شود، مقادیر جدید تولید می‌شود
3. **دو روش برای کلیک دکمه:**
   - **با متن:** `click: عضویت` - برای دکمه‌های ثابت
   - **با شماره:** `click: #0` یا `click: 0` - برای دکمه‌های متغیر
4. **شماره دکمه‌ها از 0 شروع می‌شود:**
   - 0 = اولین دکمه
   - 1 = دومین دکمه
   - 2 = سومین دکمه
5. **فوروارد برای جمع‌آوری نتایج:**
   - معمولاً در آخر سناریو استفاده می‌شود
   - فرمت: `forward: تعداد, @هدف`
   - مثال: `forward: 5, @mychannel`
6. **جستجوی دکمه جزئی است** - برای کلیک کافی است بخشی از متن دکمه را بنویسید
7. **بین هر مرحله 2 ثانیه صبر می‌کند** - می‌توانید با `wait` این زمان را تغییر دهید
8. **می‌توانید چند متغیر در یک خط استفاده کنید**:
   ```
   send: من {random:5} هستم و کد من {random_num:4} است
   ```

## خطاهای رایج

### ❌ فراموش کردن یوزرنیم ربات
```
start: ref123
send: سلام
```
**درست:**
```
@MyBot
start: ref123
send: سلام
```

### ❌ استفاده از متغیر بدون مقدار
```
send: سلام {random} هستم
```
**درست:**
```
send: سلام {random:5} هستم
```

### ❌ فراموش کردن دو نقطه بعد از دستور
```
send سلام
```
**درست:**
```
send: سلام
```

## کاربرد فوروارد (Forward)

### چرا از فوروارد استفاده کنیم؟

وقتی سناریو را روی چندین اکانت اجرا می‌کنید، ربات برای هر اکانت نتیجه‌ای می‌دهد (مثل لینک، کد، پیام تایید و غیره). با استفاده از `forward` می‌توانید همه این نتایج را در یک کانال جمع‌آوری کنید.

### مثال واقعی

فرض کنید ربات بعد از تکمیل کار، یک لینک اختصاصی به شما می‌دهد:

```
@LinkGeneratorBot
start: ref123
click: ساخت لینک
send: نام من {random:6} است
click: #0
# ربات 2 پیام می‌فرستد: یک پیام تایید + یک لینک
forward: 2, @my_links_channel
```

**نتیجه:** همه لینک‌های تولید شده توسط همه اکانت‌ها در کانال `@my_links_channel` جمع می‌شوند!

### نکات مهم

1. **تعداد پیام:** دقت کنید چند پیام می‌خواهید فوروارد کنید
   - اگر فقط نتیجه نهایی مهم است: `forward: 1, @channel`
   - اگر چند پیام مهم است: `forward: 5, @channel`

2. **هدف باید از قبل عضو باشید:** اکانت‌ها باید عضو کانال/گروه هدف باشند

3. **ترتیب پیام‌ها:** پیام‌ها از قدیمی به جدید فوروارد می‌شوند

### مثال‌های مختلف

**فوروارد فقط نتیجه نهایی:**
```
forward: 1, @results
```

**فوروارد چند پیام:**
```
forward: 5, @all_messages
```

**فوروارد به گروه:**
```
forward: 3, @my_group
```

### چرا از شماره دکمه استفاده کنیم؟

بعضی ربات‌ها برای هر کاربر دکمه‌های متفاوتی نمایش می‌دهند. مثلاً:
- برای اکانت 1: دکمه "کانال A"
- برای اکانت 2: دکمه "کانال B"
- برای اکانت 3: دکمه "کانال C"

در این حالت نمی‌توانیم از `click: کانال` استفاده کنیم چون متن دکمه‌ها متفاوت است.

**راه حل:** از شماره دکمه استفاده کنید:
```
click: #0    # همیشه اولین دکمه را کلیک می‌کند
```

### مثال واقعی

فرض کنید ربات برای هر کاربر یک کانال تصادفی نمایش می‌دهد:

```
@RandomChannelBot
start: ref123
# ربات 3 دکمه نمایش می‌دهد (هر بار متفاوت)
click: #0    # کلیک اولین دکمه (هر چی باشه)
wait: 3
click: #1    # کلیک دومین دکمه
wait: 3
click: #2    # کلیک سومین دکمه
click: تایید    # کلیک دکمه تایید (ثابت)
```

قبل از اجرای دسته‌جمعی، سناریو خود را با یک اکانت تست کنید تا مطمئن شوید همه مراحل درست کار می‌کند.

## پشتیبانی

اگر سوالی دارید یا به مشکلی برخوردید، با ادمین تماس بگیرید.
