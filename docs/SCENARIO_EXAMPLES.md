## توضیحات کلی

سناریو پیشرفته به شما امکان می‌دهد یک یا چند ربات را با سناریوهای مختلف به صورت خودکار اجرا کنید.

### ✨ قابلیت‌های اصلی:

- ✅ **اجرای خودکار**: سناریوها به صورت کاملاً خودکار اجرا می‌شوند
- ✅ **چند ربات**: می‌توانید چندین ربات را در یک سناریو اجرا کنید
- ✅ **رفرال چندگانه**: تقسیم اکانت‌ها بین چند کد رفرال
- ✅ **متغیرهای دینامیک**: تولید خودکار رشته‌های تصادفی
- ✅ **حل خودکار کپچا**: حل معادلات ریاضی به صورت خودکار
- ✅ **فوروارد هوشمند**: فوروارد پیام‌های خاص به کانال
- ✅ **کنترل اجرا**: توقف موقت، ادامه، و لغو
- ✅ **گزارش کامل**: گزارش دقیق از همه عملیات
- ✅ **یادداشت‌گذاری**: ثبت یادداشت برای هر ربات

### 🎮 نحوه کار:

1. شما سناریو را می‌نویسید
2. تعداد اکانت را انتخاب می‌کنید
3. سرعت اجرا را تعیین می‌کنید
4. ربات به صورت خودکار اجرا می‌کند
5. گزارش کامل دریافت می‌کنید
6. یادداشت ثبت می‌کنید (اختیاری)

## دستورات موجود

### 1. `start: ref_id`
**استارت ربات با پارامتر رفرال**

```
start: AAAAACWiOtQ
```

**کاربرد:**
- ارسال دستور `/start` به ربات
- استفاده برای لینک‌های رفرال
- اولین دستور در اکثر سناریوها

**نکته:** اگر رفرال ندارید، خالی بگذارید:
```
start:
```

---

### 2. `send: متن`
**ارسال پیام به ربات**

```
send: سلام من کاربر جدید هستم
```

**کاربرد:**
- ارسال متن به ربات
- ارسال دستورات (مثل `/help`)
- ارسال اطلاعات (مثل یوزرنیم)

**مثال‌ها:**
```
send: /start
send: نام من علی است
send: یوزرنیم من {random:8} است
```

---

### 3. `click: کلمه کلیدی` یا `click: #N`
**کلیک روی دکمه**

#### روش 1: جستجوی متن (برای دکمه‌های ثابت)
```
click: عضویت
click: تایید
click: ادامه
```

**کاربرد:**
- وقتی متن دکمه ثابت است
- جستجو case-insensitive (حروف بزرگ/کوچک مهم نیست)
- جستجوی جزئی (partial match)

#### روش 2: شماره دکمه (برای دکمه‌های متغیر)
```
click: #0    # اولین دکمه
click: #1    # دومین دکمه
click: 2     # سومین دکمه (بدون # هم کار می‌کند)
```

**کاربرد:**
- وقتی متن دکمه‌ها برای هر اکانت متفاوت است
- وقتی نمی‌دانید متن دکمه چیست
- برای دکمه‌های دینامیک

💡 **نکته:** شماره دکمه‌ها از 0 شروع می‌شود:
- دکمه اول = 0
- دکمه دوم = 1
- دکمه سوم = 2

**مثال واقعی:**
```
@RandomChannelBot
start: ref123
# ربات 3 دکمه نمایش می‌دهد (هر بار متفاوت)
click: #0    # کلیک اولین دکمه (هر چی باشه)
wait: 3
click: #1    # کلیک دومین دکمه
click: تایید    # کلیک دکمه تایید (ثابت)
```

### 4. `join: لینک`
**جوین کانال یا گروه**

```
join: https://t.me/mychannel
join: @mychannel
join: https://t.me/+ABC123xyz
```

**فرمت‌های پشتیبانی شده:**
- لینک عمومی: `https://t.me/channel_name`
- یوزرنیم: `@channel_name`
- لینک خصوصی: `https://t.me/+hash` یا `https://t.me/joinchat/hash`

**کاربرد:**
- عضویت در کانال‌های اجباری
- جوین گروه‌ها
- عضویت در کانال‌های خصوصی

---

### 5. `leave: لینک`
**لفت کانال یا گروه**

```
leave: https://t.me/mychannel
leave: @mychannel
```

**کاربرد:**
- خروج از کانال بعد از تکمیل کار
- لفت گروه‌های موقت
- مدیریت عضویت‌ها

---

### 6. `wait: ثانیه`
**صبر کردن**

```
wait: 5
wait: 3
wait: 10
```

**کاربرد:**
- صبر برای دریافت پاسخ ربات
- تاخیر بین عملیات
- زمان برای پردازش

**تفاوت با `stop`:**
- `wait`: صبر معمولی بین مراحل
- `stop`: توقف موقت (برای مثال بعد از جوین کانال)

---

### 7. `stop: ثانیه`
**توقف موقت سناریو**

```
stop: 5    # توقف 5 ثانیه
stop: 3    # توقف 3 ثانیه
stop:      # توقف پیش‌فرض 5 ثانیه
```

**کاربرد:**
- توقف بعد از جوین کانال
- زمان برای بررسی عضویت
- تاخیر قبل از ادامه کار

**مثال:**
```
join: https://t.me/channel1
stop: 5    # صبر 5 ثانیه تا عضویت تایید شه
click: بررسی عضویت
```

### 8. `solve_captcha: send` یا `solve_captcha: click`
**حل خودکار کپچای ریاضی**

#### فرمت 1: ارسال جواب به صورت متن
```
solve_captcha: send
```

#### فرمت 2: کلیک روی دکمه با جواب
```
solve_captcha: click
```

#### فرمت 3: بررسی چند پیام آخر
```
solve_captcha: send, 3    # بررسی 3 پیام آخر
solve_captcha: click, 5   # بررسی 5 پیام آخر
```

**معادلات پشتیبانی شده:**
- جمع: `5 + 3 = ?` → 8
- تفریق: `81 - 4 = ?` → 77
- ضرب: `5 × 3 = ?` → 15
- تقسیم: `10 ÷ 2 = ?` → 5

**کاربرد:**
- حل خودکار کپچاهای ریاضی
- جلوگیری از توقف سناریو
- افزایش نرخ موفقیت

**مثال:**
```
@CaptchaBot
start: ref123
solve_captcha: send    # ربات کپچا میفرسته، خودکار حل میشه
click: ادامه
```

### 10. `forward: N, @target` یا `forward: "متن", @target`
**فوروارد پیام‌های ربات**

#### روش 1: بر اساس تعداد (ساده)
```
forward: 5, @mychannel        # فوروارد 5 پیام آخر
forward: 3, @results_group    # فوروارد 3 پیام آخر
forward: 1, mychannel         # فوروارد 1 پیام آخر (بدون @ هم کار می‌کند)
```

#### روش 2: بر اساس متن (هوشمند - جدید!)
```
forward: "لینک شما", @mychannel           # فوروارد پیام‌هایی که شامل "لینک شما" هستند
forward: "کد تایید", @results             # فوروارد پیام‌هایی که شامل "کد تایید" هستند
forward: 'https://', @links_channel       # فوروارد پیام‌هایی که شامل لینک هستند
```

**تفاوت دو روش:**

**روش قدیم (بر اساس تعداد):**
```
@LinkBot
start: ref123
send: /getlink
# ربات 3 پیام می‌فرسته:
# 1. "درخواست شما دریافت شد"
# 2. "در حال پردازش..."
# 3. "لینک شما: https://..."

forward: 3, @channel    # هر 3 پیام فوروارد میشه (شلوغ!)
```

**روش جدید (بر اساس متن):**
```
@LinkBot
start: ref123
send: /getlink
# فقط پیام حاوی لینک فوروارد میشه
forward: "لینک شما:", @channel    # فقط پیام 3 فوروارد میشه! ✅
```

**نکات:**
- فرمت 1: N تا پیام آخر را فوروارد می‌کند
- فرمت 2: در 100 پیام آخر جستجو می‌کند و همه پیام‌های حاوی متن مشخص را فوروارد می‌کند
- جستجو case-insensitive است (حروف بزرگ/کوچک مهم نیست)
- معمولاً در آخر سناریو استفاده می‌شود

**کاربرد:**
- جمع‌آوری نتایج همه اکانت‌ها در یک کانال
- ذخیره لینک‌ها، کدها، پیام‌های مهم
- آرشیو نتایج

## متغیرهای دینامیک

### `{random:N}`
**رشته تصادفی N حرفی (حروف کوچک + اعداد)**

```
send: یوزرنیم من {random:8} است
```

**خروجی:**
- اکانت 1: `یوزرنیم من abc12xyz است`
- اکانت 2: `یوزرنیم من def34uvw است`

**کاربرد:**
- تولید یوزرنیم یونیک
- ارسال اطلاعات متفاوت برای هر اکانت
- جلوگیری از تکراری بودن

---

### `{random_upper:N}`
**رشته تصادفی N حرفی (حروف بزرگ + اعداد)**

```
send: کد من {random_upper:6} است
```

**خروجی:**
- اکانت 1: `کد من ABC123 است`
- اکانت 2: `کد من XYZ789 است`

**کاربرد:**
- کدهای بزرگ
- شناسه‌های خاص
- رمزهای موقت

---

### `{random_num:N}`
**عدد تصادفی N رقمی**

```
send: شماره من {random_num:4} است
```

**خروجی:**
- اکانت 1: `شماره من 1234 است`
- اکانت 2: `شماره من 5678 است`

**کاربرد:**
- تولید شماره
- کدهای عددی
- شناسه‌های رقمی

**نکته:** می‌توانید چند متغیر در یک خط استفاده کنید:
```
send: من {random:5} هستم و کد من {random_num:4} است
```

## رفرال چندگانه

### 🎯 قابلیت جدید: تقسیم اکانت‌ها بین چند کد رفرال

**فرمت:**
```
@bot_username
start: ref_code_1 | 20
start: ref_code_2 | 30
start: ref_code_3 | 50
send: متن
click: دکمه
```

### نحوه کار:

1. **شروع از کد اول:**
   - سیستم از `ref_code_1` شروع می‌کند
   - تا 20 تا **موفق** نشود، به کد بعدی نمی‌رود

2. **شمارش فقط موفق‌ها:**
   - اگر از 20 تا فقط 15 تا موفق شد
   - 5 تای دیگه از اکانت‌های بعدی استفاده می‌شود
   - تا 20 تا موفق کامل شود

3. **رفتن به کد بعدی:**
   - بعد از 20 تا موفق، می‌رود سراغ `ref_code_2`
   - همین روند تا 30 تا موفق ادامه می‌یابد

4. **ادامه تا پایان:**
   - بعد می‌رود سراغ `ref_code_3` تا 50 تا موفق شود
   - سیستم خودکار بین کدها جابجا می‌شود

### مثال واقعی:

```
@MyReferralBot
start: team_A | 50
start: team_B | 50
start: team_C | 50
send: /start
click: عضویت
join: https://t.me/channel
click: تایید
```

**نتیجه:**
- 50 تا موفق برای team_A
- 50 تا موفق برای team_B
- 50 تا موفق برای team_C
- جمع: 150 تا موفق

### گزارش نهایی:

```
📊 آمار رفرال:
━━━━━━━━━━━━━━━━━━━━

🤖 @MyReferralBot
  ✅ team_A: 50/50 موفق (8 ناموفق) | 58 اکانت
  ✅ team_B: 50/50 موفق (12 ناموفق) | 62 اکانت
  ✅ team_C: 50/50 موفق (15 ناموفق) | 65 اکانت

📱 جمع اکانت‌های استفاده شده: 185
✅ جمع موفق: 150
❌ جمع ناموفق: 35
```

### مزایا:

✅ **تقسیم عادلانه**: هر کد رفرال دقیقاً تعداد مورد نظر را می‌گیرد
✅ **شمارش دقیق**: فقط موفق‌ها شمارش می‌شوند
✅ **خودکار**: سیستم خودکار بین کدها جابجا می‌شود
✅ **گزارش کامل**: آمار دقیق هر کد رفرال
✅ **بهینه**: از اکانت‌ها به صورت بهینه استفاده می‌شود

### چند ربات با رفرال چندگانه:

```
@Bot1
start: ref1_A | 10
start: ref1_B | 15
send: سلام
click: #0

@Bot2
start: ref2_A | 20
start: ref2_B | 25
send: /start
click: تایید
```

**نتیجه:**
- هر ربات رفرال‌های خودش را مدیریت می‌کند
- گزارش جداگانه برای هر ربات
- آمار دقیق برای هر کد رفرال

## انتخاب محدوده اکانت‌ها

### دستورات موجود

وقتی سناریو را ارسال می‌کنید، ربات از شما می‌پرسد چند اکانت استفاده شود:

#### 1. عدد ساده
```
5
```
5 اکانت اول را اجرا می‌کند (اکانت 1 تا 5)

#### 2. همه اکانت‌ها
```
/all
```
همه اکانت‌های فعال را اجرا می‌کند

#### 3. شروع از شماره خاص
```
/from 70
```
از اکانت 70 تا آخر اجرا می‌کند

#### 4. محدوده مشخص
```
/from 70 to 100
```
فقط اکانت‌های 70 تا 100 را اجرا می‌کند

### مثال‌های کاربردی

#### مثال 1: تست روی چند اکانت
```
مرحله 1: سناریو را ارسال کنید
ربات: چند تا اکانت؟
شما: 3

نتیجه: اکانت‌های 1، 2، 3 اجرا می‌شوند ✅

مرحله 2: بعد از تست موفق
شما: همان سناریو را دوباره ارسال کنید
ربات: چند تا اکانت؟
شما: /from 4

نتیجه: از اکانت 4 تا آخر اجرا می‌شوند ✅
```

#### مثال 2: تقسیم کار بین چند سرور
```
سرور 1:
شما: /from 1 to 100
نتیجه: اکانت‌های 1 تا 100 ✅

سرور 2:
شما: /from 101 to 200
نتیجه: اکانت‌های 101 تا 200 ✅

سرور 3:
شما: /from 201
نتیجه: از اکانت 201 تا آخر ✅
```

## فرمت‌های پشتیبانی شده

### 1. سناریو یک ربات

```
@bot_username
start: ref_id
send: متن
click: دکمه
```

### 2. سناریو چند ربات

```
@bot1
start: ref1
send: text1

@bot2
start: ref2
click: #0
```

### 3. سناریو با رفرال چندگانه (جدید! 🎉)

```
@bot_username
start: ref_code_1 | 20
start: ref_code_2 | 30
start: ref_code_3 | 50
send: متن
click: دکمه
```

**توضیح:**
- `ref_code_1 | 20` = 20 تا موفق برای کد رفرال اول
- `ref_code_2 | 30` = 30 تا موفق برای کد رفرال دوم
- `ref_code_3 | 50` = 50 تا موفق برای کد رفرال سوم

**نکات مهم:**
- فقط اکانت‌های **موفق** شمارش می‌شوند
- اگر از 20 تا فقط 15 تا موفق شد، 5 تای دیگه از اکانت‌های بعدی استفاده می‌شه
- سیستم خودکار بین کدهای رفرال جابجا می‌شه تا همه به هدف برسند
- در گزارش نهایی، آمار دقیق هر کد رفرال نمایش داده می‌شود

💡 **نکته:** بین رباتها یک خط خالی بگذارید (اختیاری ولی خوانا‌تر است)

## مثال‌های کامل

1. **تست قبل از اجرای دسته‌جمعی:** همیشه سناریو را با 1-2 اکانت تست کنید
2. **بررسی دکمه‌ها:** اگر متن دکمه‌ها متغیر است، از شماره دکمه استفاده کنید
3. **زمان کافی:** بین مراحل زمان کافی بگذارید (wait یا stop)
4. **فوروارد هوشمند:** از فوروارد بر اساس متن برای نتایج دقیق‌تر استفاده کنید

## نکات مهم

1. **یک یا چند ربات:** می‌توانید یک ربات یا چند ربات را در یک سناریو اجرا کنید
2. **شروع با @:** هر ربات باید با @ شروع شود
3. **جدا کردن رباتها:** بین رباتها یک خط خالی بگذارید (اختیاری)
4. **متغیرها برای هر اکانت متفاوت هستند** - هر بار که سناریو اجرا می‌شود، مقادیر جدید تولید می‌شود
5. **دو روش برای کلیک دکمه:**
   - **با متن:** `click: عضویت` - برای دکمه‌های ثابت
   - **با شماره:** `click: #0` یا `click: 0` - برای دکمه‌های متغیر
6. **شماره دکمه‌ها از 0 شروع می‌شود:**
   - 0 = اولین دکمه
   - 1 = دومین دکمه
   - 2 = سومین دکمه
7. **فوروارد برای جمع‌آوری نتایج:**
   - معمولاً در آخر سناریو استفاده می‌شود
   - فرمت: `forward: تعداد, @هدف`
   - مثال: `forward: 5, @mychannel`
8. **ترتیب اجرا:** رباتها به ترتیب اجرا می‌شوند (ربات 1، سپس ربات 2، و غیره)
9. **تاخیر بین رباتها:** بین هر ربات 2 ثانیه صبر می‌کند
10. **می‌توانید چند متغیر در یک خط استفاده کنید**:
   ```
   send: من {random:5} هستم و کد من {random_num:4} است
   ```

## کاربرد فوروارد (Forward)

### دو روش فوروارد

#### روش 1: بر اساس تعداد (ساده)
```
forward: 5, @channel    # 5 پیام آخر
```

#### روش 2: بر اساس متن (هوشمند - جدید!)
```
forward: "لینک شما", @channel    # فقط پیام‌های حاوی "لینک شما"
```

### چرا از فوروارد استفاده کنیم؟

وقتی سناریو را روی چندین اکانت اجرا می‌کنید، ربات برای هر اکانت نتیجه‌ای می‌دهد (مثل لینک، کد، پیام تایید و غیره). با استفاده از `forward` می‌توانید همه این نتایج را در یک کانال جمع‌آوری کنید.

### مقایسه دو روش

**مشکل روش قدیم (بر اساس تعداد):**
```
@LinkBot
start: ref123
send: /getlink
# ربات 3 پیام می‌فرسته:
# 1. "درخواست شما دریافت شد"
# 2. "در حال پردازش..."
# 3. "لینک شما: https://..."

forward: 3, @channel    # هر 3 پیام فوروارد میشه (شلوغ!)
```

**راه‌حل جدید (بر اساس متن):**
```
@LinkBot
start: ref123
send: /getlink
# فقط پیام حاوی لینک فوروارد میشه
forward: "لینک شما:", @channel    # فقط پیام 3 فوروارد میشه! ✅
```

### مثال واقعی - روش قدیم

فرض کنید ربات بعد از تکمیل کار، یک لینک اختصاصی به شما می‌دهد:

```
@LinkGeneratorBot
start: ref123
click: ساخت لینک
send: نام من {random:6} است
click: #0
# ربات 2 پیام می‌فرستد: یک پیام تایید + یک لینک
forward: 2, @my_links_channel
```

**نتیجه:** همه لینک‌های تولید شده توسط همه اکانت‌ها در کانال `@my_links_channel` جمع می‌شوند!

### مثال واقعی - روش جدید (بهتر!)

```
@LinkGeneratorBot
start: ref123
click: ساخت لینک
send: نام من {random:6} است
click: #0
# فقط پیام حاوی لینک فوروارد میشه
forward: "https://", @my_links_channel
```

**نتیجه:** فقط لینک‌ها فوروارد میشن، نه پیام‌های اضافی! 🎯

### نکات مهم

1. **روش بر اساس تعداد:**
   - ساده و سریع
   - برای زمانی که تعداد پیام‌ها ثابته
   - ممکنه پیام‌های اضافی هم فوروارد بشه

2. **روش بر اساس متن (جدید):**
   - دقیق و هوشمند
   - فقط پیام مورد نظر فوروارد میشه
   - در 100 پیام آخر جستجو می‌کنه
   - جستجو case-insensitive (حروف بزرگ/کوچک مهم نیست)

3. **هدف باید از قبل عضو باشید:** اکانت‌ها باید عضو کانال/گروه هدف باشند

4. **ترتیب پیام‌ها:** پیام‌ها از قدیمی به جدید فوروارد می‌شوند

### مثال‌های کاربردی بیشتر

**فوروارد کدهای تایید:**
```
forward: "کد تایید", @codes
```

**فوروارد فقط لینک‌ها:**
```
forward: "https://", @links
forward: "http://", @links
```

**فوروارد پیام موفقیت:**
```
forward: "تبریک", @success_messages
forward: "موفق", @results
```

**فوروارد اطلاعات خاص:**
```
forward: "یوزرنیم شما", @usernames
forward: "رمز عبور", @passwords
forward: "توکن", @tokens
```

### مثال‌های مختلف

**فوروارد فقط نتیجه نهایی:**
```

```

**فوروارد چند پیام:**
```
forward: 5, @all_messages
```

**فوروارد به گروه:**
```

```

### چرا از شماره دکمه استفاده کنیم؟

بعضی ربات‌ها برای هر کاربر دکمه‌های متفاوتی نمایش می‌دهند. مثلاً:
- برای اکانت 1: دکمه "کانال A"
- برای اکانت 2: دکمه "کانال B"
- برای اکانت 3: دکمه "کانال C"

در این حالت نمی‌توانیم از `click: کانال` استفاده کنیم چون متن دکمه‌ها متفاوت است.

**راه حل:** از شماره دکمه استفاده کنید:
```
click: #0    # همیشه اولین دکمه را کلیک می‌کند
```

### مثال واقعی

فرض کنید ربات برای هر کاربر یک کانال تصادفی نمایش می‌دهد:

```
@RandomChannelBot

# ربات 3 دکمه نمایش می‌دهد (هر بار متفاوت)

wait: 3
click: #1    # کلیک دومین دکمه
wait: 3
click: #2    # کلیک سومین دکمه
click: تایید    # کلیک دکمه تایید (ثابت)
```

قبل از اجرای دسته‌جمعی، سناریو خود را با یک اکانت تست کنید تا مطمئن شوید همه مراحل درست کار می‌کند.

## انتخاب محدوده اکانت‌ها

### دستورات موجود

وقتی سناریو را ارسال می‌کنید، ربات از شما می‌پرسد چند اکانت استفاده شود. گزینه‌های زیر در دسترس است:

#### 1. عدد ساده
```
5
```
5 اکانت اول را اجرا می‌کند (اکانت 1 تا 5)

#### 2. همه اکانت‌ها
```
/all
```
همه اکانت‌های فعال را اجرا می‌کند

#### 3. شروع از شماره خاص
```
/from 70
```
از اکانت 70 تا آخر اجرا می‌کند

#### 4. محدوده مشخص
```
/from 70 to 100
```
فقط اکانت‌های 70 تا 100 را اجرا می‌کند

### مثال‌های کاربردی

#### مثال 1: تست روی چند اکانت
```
مرحله 1: سناریو را ارسال کنید
ربات: چند تا اکانت؟
شما: 3

نتیجه: اکانت‌های 1، 2، 3 اجرا می‌شوند ✅

مرحله 2: بعد از تست موفق
شما: همان سناریو را دوباره ارسال کنید
ربات: چند تا اکانت؟
شما: /from 4

نتیجه: از اکانت 4 تا آخر اجرا می‌شوند ✅
```

#### مثال 2: تقسیم کار بین چند سرور
```
سرور 1:
شما: /from 1 to 100
نتیجه: اکانت‌های 1 تا 100 ✅

سرور 2:
شما: /from 101 to 200
نتیجه: اکانت‌های 101 تا 200 ✅

سرور 3:
شما: /from 201
نتیجه: از اکانت 201 تا آخر ✅
```

#### مثال 3: اجرای مجدد بخش خاص
```
فرض کنید اکانت‌های 50 تا 80 با خطا مواجه شدند:

شما: همان سناریو را ارسال کنید
ربات: چند تا اکانت؟
شما: /from 50 to 80

نتیجه: فقط اکانت‌های 50 تا 80 دوباره اجرا می‌شوند ✅
```

#### مثال 4: ادامه بعد از قطع برق
```
شما 500 اکانت دارید:

روز اول:
شما: /from 1 to 200
نتیجه: اکانت‌های 1 تا 200 اجرا شدند ✅

روز دوم:
شما: /from 201 to 400
نتیجه: اکانت‌های 201 تا 400 اجرا شدند ✅

روز سوم:
شما: /from 401
نتیجه: اکانت‌های 401 تا 500 اجرا شدند ✅
```

### نکات مهم

1. **شماره‌گذاری از 1 شروع می‌شود** نه صفر
   - اکانت اول = شماره 1
   - اکانت دوم = شماره 2
   - و غیره...

2. **محدوده `to` شامل عدد پایانی است**
   - `/from 70 to 100` = اکانت‌های 70، 71، ..., 99، 100

3. **اعتبارسنجی خودکار**
   - اگر عدد بیشتر از تعداد اکانت‌ها باشد، خطا می‌دهد
   - اگر عدد شروع بیشتر از عدد پایان باشد، خطا می‌دهد

4. **ترکیب با Resume**
   - اگر سناریو قبلاً شروع شده باشد، می‌توانید با `/from` از جای دیگری شروع کنید
   - مثلاً: پیشرفت قبلی در اکانت 70 بود، ولی شما می‌توانید `/from 50` بزنید

### خطاهای رایج

#### ❌ فراموش کردن کلمه `from`
```

```
این 70 اکانت اول را اجرا می‌کند، نه از اکانت 70!

✅ درست:
```
/from 70
```

#### ❌ فراموش کردن کلمه `to`
```
/from 70 100
```
خطا می‌دهد!

✅ درست:
```

```

#### ❌ استفاده از اعداد منفی
```

```
خطا می‌دهد!

✅ درست:
```

```
