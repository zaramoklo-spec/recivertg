# 📝 تغییرات: قابلیت یادداشت‌گذاری برای رباتها

## تاریخ: 2026-05-04

## خلاصه تغییرات

یک سیستم کامل یادداشت‌گذاری برای رباتها اضافه شد که به کاربران امکان می‌دهد بعد از اجرای هر سناریو، یادداشت‌های شخصی برای هر ربات ثبت کنند.

## فایل‌های جدید

### 1. `src/services/note_manager.py`
- کلاس `NoteManager` برای مدیریت یادداشت‌ها
- متدهای `add_note`, `get_user_notes`, `get_bot_notes`, `delete_note`, `update_note`

### 2. `docs/NOTES_FEATURE.md`
- مستندات کامل قابلیت یادداشت
- راهنمای استفاده
- مثال‌های کاربردی
- سوالات متداول

### 3. `CHANGELOG_NOTES.md`
- این فایل!

## تغییرات در فایل‌های موجود

### 1. `src/database/models.py`

#### جدول جدید:
```sql
CREATE TABLE bot_notes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    bot_username TEXT NOT NULL,
    note_text TEXT NOT NULL,
    scenario_text TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users (user_id)
)
```

#### متدهای جدید:
- `add_bot_note()` - افزودن یادداشت
- `get_user_notes()` - دریافت همه یادداشت‌های کاربر
- `get_bot_notes()` - دریافت یادداشت‌های یک ربات خاص
- `delete_note()` - حذف یادداشت
- `update_note()` - ویرایش یادداشت

### 2. `src/services/__init__.py`
- اضافه شدن `NoteManager` به exports

### 3. `src/bot/handlers.py`

#### تغییرات در منوی اصلی:
- اضافه شدن دکمه **📝 یادداشت‌های من** برای سازنده و ادمین‌ها

#### Handler های جدید:

**Callback Handlers:**
- `my_notes_callback` - نمایش یادداشت‌های کاربر
- `note_yes_callback` - کاربر می‌خواهد یادداشت بزند
- `note_no_callback` - کاربر نمی‌خواهد یادداشت بزند (تک ربات)
- `note_skip_callback` - رد کردن یادداشت فعلی (چند ربات)
- `note_no_all_callback` - عدم ثبت یادداشت برای همه رباتها
- `delete_note_menu_callback` - منوی حذف یادداشت

**Command Handlers:**
- `/notes @bot_username` - مشاهده یادداشت‌های یک ربات
- `/deletenote NOTE_ID` - حذف یادداشت
- `/editnote NOTE_ID` - ویرایش یادداشت

**Message Handler Steps:**
- `waiting_note_single` - دریافت متن یادداشت برای تک ربات
- `waiting_note_multi` - دریافت متن یادداشت برای چند ربات
- `edit_note` - دریافت متن جدید برای ویرایش

#### تغییرات در جریان سناریو:
بعد از ارسال گزارش کامل سناریو، سیستم از کاربر می‌پرسد:

**برای تک ربات:**
```
📝 یادداشت برای ربات @BotName

آیا می‌خواهید یادداشتی برای این ربات ثبت کنید؟

[✅ بله، یادداشت می‌زنم]
[❌ نه، نیازی نیست]
```

**برای چند ربات:**
```
📝 یادداشت برای ربات @Bot1

آیا می‌خواهید یادداشتی برای این ربات ثبت کنید؟

[✅ بله، یادداشت می‌زنم]
[⏭ بعدی]
[❌ نه، برای هیچکدام]
```

### 4. `README.md`
- اضافه شدن قابلیت یادداشت به لیست ویژگی‌ها

## جریان کار (Workflow)

### 1. اجرای سناریو
```
کاربر → سناریو پیشرفته → اجرا → گزارش
```

### 2. سوال یادداشت
```
گزارش → سوال یادداشت → [بله/خیر]
```

### 3. ثبت یادداشت (اگر بله)
```
بله → ارسال متن → ذخیره در DB → تایید
```

### 4. چند ربات
```
ربات 1 → یادداشت؟ → [بله/بعدی/خیر برای همه]
  ↓
ربات 2 → یادداشت؟ → [بله/بعدی/خیر برای همه]
  ↓
...
```

## مزایا

✅ **سازماندهی بهتر**: کاربران می‌توانند اطلاعات مهم درباره هر ربات را ذخیره کنند
✅ **یادآوری**: نکات مهم، لینک‌های رفرال، تنظیمات خاص را فراموش نمی‌کنند
✅ **تاریخچه**: نتایج قبلی و تجربیات را ثبت کنند
✅ **بهینه‌سازی**: بر اساس یادداشت‌های قبلی، سناریوها را بهبود دهند
✅ **اشتراک‌گذاری**: ادمین‌ها می‌توانند یادداشت‌های خود را با هم به اشتراک بگذارند

## نکات فنی

### 1. دسترسی
- فقط سازنده و ادمین‌ها می‌توانند یادداشت ثبت کنند
- هر کاربر فقط یادداشت‌های خودش را می‌بیند

### 2. ذخیره‌سازی
- یادداشت‌ها در دیتابیس SQLite ذخیره می‌شوند
- متن سناریو همراه با یادداشت ذخیره می‌شود (اختیاری)
- تاریخ ایجاد و ویرایش ثبت می‌شود

### 3. مدیریت State
- از `user_states` برای مدیریت جریان یادداشت استفاده می‌شود
- برای چند ربات، `current_bot_index` پیشرفت را ردیابی می‌کند

### 4. Logging
- همه عملیات یادداشت در جدول `stats` لاگ می‌شوند
- شامل: `add_note`, `delete_note`, `edit_note`

## تست

برای تست قابلیت:

1. یک سناریو تک ربات اجرا کنید
2. بعد از گزارش، روی "بله، یادداشت می‌زنم" کلیک کنید
3. یک یادداشت ارسال کنید
4. با `/notes @bot_username` یادداشت را مشاهده کنید
5. با `/editnote ID` یادداشت را ویرایش کنید
6. با `/deletenote ID` یادداشت را حذف کنید

برای چند ربات:
1. یک سناریو چند ربات اجرا کنید
2. برای هر ربات یادداشت ثبت کنید یا رد کنید
3. یادداشت‌ها را مشاهده کنید

## Migration

هنگام اجرای ربات، جدول `bot_notes` به صورت خودکار ایجاد می‌شود.
نیازی به migration دستی نیست.
